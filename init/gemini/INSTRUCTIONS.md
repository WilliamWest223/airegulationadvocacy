# Gemini Reporting Instructions

Use this folder for all Gemini-specific work logs.

## Required behavior

1. For each meaningful task, create a report markdown file in `reports/`.
2. Use filename format: `YYYY-MM-DD-brief-topic.md`.
3. Keep notes concise but complete enough for handoff and review.

## Suggested report template

```md
# Task Report: <brief topic>

## Date
YYYY-MM-DD

## Objective
What was requested.

## Actions Taken
- Key action 1
- Key action 2

## Files Touched
- `path/to/file`

## Outcome
What changed and current status.

## Follow-ups
- Optional next steps
```
